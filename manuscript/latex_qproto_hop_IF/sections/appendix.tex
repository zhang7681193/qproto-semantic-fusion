\section{Implementation Notes}

The HOP branch is applied to schema-adapted readout values rather than to the value-and-mask vector. This avoids turning missingness masks into high-order class features. The low-order branch still receives both values and masks, so missing observables remain distinguishable from zero-valued expectations.

The reported communication is computed as the number of transmitted scalar statistics times four bytes. The count includes class prototypes, class counts, high-order sketches when used, and small metadata fields. Public schema definitions and anchors are treated as protocol setup objects rather than per-round payloads.

\section{Additional Theory Details}

\subsection{Plug-In Classification Consistency}

Let \(G(z)=g(A_{\schema}(r(x)))\) denote the protocol feature used by the classifier, where \(g\) may contain low-order RFF features, coverage-aware values, and HOP features. Define the population class statistic
\begin{equation}
    \theta_c^\star=\E[G(z)\mid y=c]
\end{equation}
and the population score
\begin{equation}
    s_c^\star(z)=\|G(z)-\theta_c^\star\|_2^2.
\end{equation}
The empirical server statistic \(\hat{\theta}_c\) is a plug-in estimator of \(\theta_c^\star\). If the margin
\begin{equation}
    \Delta(z)=\min_{c\neq y}\{s_c^\star(z)-s_y^\star(z)\}
\end{equation}
is positive and
\begin{equation}
    \max_c | \hat{s}_c(z)-s_c^\star(z)| < \Delta(z)/2,
\end{equation}
then the empirical classifier returns the same label as the population nearest-prototype classifier. This is the margin condition used implicitly in the finite-budget theorem in the main paper. It converts estimation error in comparable readout statistics into a classification guarantee.

\subsection{Finite-Shot Constants}

For a Pauli observable with outcomes in \(\{-1,+1\}\), Hoeffding's inequality gives
\begin{equation}
    \Pr(|\hat{\mu}_{ia}-\mu_{ia}|>\epsilon)
    \le 2\exp(-S_i\epsilon^2/2).
\end{equation}
After a union bound over \(D\) public observables, with probability at least \(1-\delta\),
\begin{equation}
    \|\hat{v}-v\|_2
    \le
    \sqrt{\frac{2D\log(2D/\delta)}{S_{\min}}}.
\end{equation}
If the feature map is \(L_g\)-Lipschitz and bounded by \(R_g\), the induced score perturbation is bounded by
\begin{equation}
    |\|\hat{G}-\theta_c\|_2^2-\|G-\theta_c\|_2^2|
    \le
    2R_gL_g\|\hat{v}-v\|_2+L_g^2\|\hat{v}-v\|_2^2.
\end{equation}
This explains why shot noise enters the finite-budget theorem through a \(D^{1/2}S_{\min}^{-1/2}\) term, while schema mismatch does not vanish with shots.

\subsection{Coverage-Aware Estimator Details}

The coverage-aware estimator stores two sufficient statistics per class and source coordinate: a value sum and an observation count. This is equivalent to estimating
\begin{equation}
    \E[V_j\mid y=c,M_j=1]
\end{equation}
instead of estimating \(\E[M_jV_j\mid y=c]\). Zero-fill prototypes estimate the latter quantity, which mixes measurement value and observation probability. This distinction is important when missingness is caused by source availability or measurement design: a missing observable should not be interpreted as a measured zero expectation. If missingness is conditionally independent of the value given the class, the coverage-aware estimator targets the full class-conditional coordinate mean. If missingness is informative, it targets the observed-view mean and the mask/coverage pattern remains available to the classifier.

\subsection{Schema-Uncertainty Details}

The main schema mismatch theorem treats a fully wrong permutation for clarity. A partially wrong schema produces the same kind of non-vanishing semantic bias. If a fraction \(\rho\) of coordinate mass is assigned to incorrect public keys, boundedness gives an \(O(\rho)\) bias term in the affected class prototypes. This term is separate from sample, shot, and random-feature errors. It can only be reduced by improving schema metadata, using calibration anchors to detect mismatches, or modeling schema uncertainty explicitly. The current method assumes protocol-defined observable identities are available; noisy schema inference is left for future work.

\subsection{Random-Feature and Communication Terms}

For bounded shift-invariant kernels, random Fourier features estimate kernel values with Monte Carlo error \(O(d^{-1/2})\) for fixed pairs. A uniform statement over \(M\) evaluated pairs follows by a union bound:
\begin{equation}
    \max_{\ell\le M}|\hat{k}(z_\ell,z'_\ell)-k(z_\ell,z'_\ell)|
    =
    O\!\left(\sqrt{\frac{\log(M/\delta)}{d}}\right).
\end{equation}
Because each class prototype uses \(d\) low-order coordinates and \(h\) high-order coordinates, the payload scales as \(C(d+h)\). Thus, increasing communication reduces sketch error but does not reduce schema error. This separation is central to the protocol claim: no amount of RFF or HOP dimension can fix an incorrect observable map.

\subsection{HOP Advantage Under Equal Means}

Suppose two classes have equal adapted readout means but covariance difference \(A=\Sigma_1-\Sigma_2\neq0\). For an isotropic Gaussian projection \(p\),
\begin{equation}
    \E_p[(p^\top A p)^2]
    =2\|A\|_F^2+(\operatorname{tr} A)^2.
\end{equation}
Therefore, whenever the class covariance matrices differ, projected quadratic features have nonzero expected squared gap even when low-order mean prototypes coincide. With \(h\) independent HOP projections, the probability that every projected gap is below a fixed detectable threshold decays exponentially in \(h\) under standard small-ball assumptions. This is the formal reason CHOP is strong on the covariance-sensing and PennyLane high-order tasks but only mildly helpful on low-order image tasks.

\subsection{Group-Balanced Source Scoring}

The group-balanced CProto rule is used when public coordinates are partitioned into semantic source groups \(G_1,\ldots,G_Q\), such as sensor blocks or feature families. Let
\begin{equation}
    \ell_{cq}(x)=
    \frac{\sum_{j\in G_q}m_j(x)\mathbf{1}\{N_{cj}>0\}w_{cj}(v_j(x)-\bar v_{cj})^2}
    {\sum_{j\in G_q}m_j(x)\mathbf{1}\{N_{cj}>0\}w_{cj}+\epsilon}
\end{equation}
be the within-group coverage-normalized class distance. Group-balanced scoring uses
\begin{equation}
    d_c^{\mathrm{grp}}(x)=|\mathcal Q_x|^{-1}\sum_{q\in\mathcal Q_x}\ell_{cq}(x),
\end{equation}
where \(\mathcal Q_x\) is the set of observed semantic groups.

\begin{proposition}[Dimensional dominance of coordinate-wise scoring]
Assume each semantic group \(G_q\) has an expected within-group loss \(L_{cq}=\E[\ell_{cq}(X)]\), and suppose a coordinate-wise distance averages over all observed coordinates without group normalization. If all coordinates in a group have comparable expected loss, the coordinate-wise score estimates a dimension-weighted objective proportional to
\begin{equation}
    \sum_q |G_q|L_{cq},
\end{equation}
whereas group-balanced scoring estimates a source-balanced objective proportional to
\begin{equation}
    \sum_q L_{cq}.
\end{equation}
Thus, when source groups have very different dimensionalities, coordinate-wise scoring can be dominated by the largest source even if the smaller sources are semantically equally important.
\end{proposition}

\begin{proof}
Ignoring the small \(\epsilon\) term and conditioning on the observed groups, coordinate-wise scoring is an average of per-coordinate squared losses over \(\cup_qG_q\). If the average coordinate loss within group \(q\) is \(L_{cq}\), then the numerator contributes approximately \(|G_q|L_{cq}\). Summing over groups gives a dimension-weighted objective. Group-balanced scoring first averages within \(G_q\) and then averages over groups, so each observed semantic source contributes one group-level term. This is the intended objective in multi-source fusion settings where a source's importance should not be determined only by how many exported coordinates it has.
\end{proof}

\subsection{Adaptive CHOP Calibration Bound}

Adaptive CHOP chooses among a finite candidate set \(\mathcal A\), containing the low-order CProto branch and a small grid of CHOP mixtures. The choice is made on a training-only calibration split and the selected candidate is then refit on the full training set. The following bound clarifies what this gate can and cannot guarantee.

\begin{proposition}[Finite calibration selection]
Let \(\mathcal A\) contain \(M\) candidate scoring rules with true calibration accuracies \(a(\alpha)\) and empirical calibration accuracies \(\hat a(\alpha)\) computed on \(n_{\mathrm{cal}}\) independent calibration examples. Let \(\hat\alpha=\arg\max_{\alpha\in\mathcal A}\hat a(\alpha)\). With probability at least \(1-\delta\),
\begin{equation}
    a(\hat\alpha)
    \ge
    \max_{\alpha\in\mathcal A}a(\alpha)
    -2\sqrt{\frac{\log(2M/\delta)}{2n_{\mathrm{cal}}}} .
\end{equation}
If the protocol requires a margin \(\tau\) over the low-order branch before enabling CHOP, the selected high-order branch has the same guarantee relative to the best candidate in \(\mathcal A\), while false high-order activation is controlled by the same uniform deviation term plus \(\tau\).
\end{proposition}

\begin{proof}
For a fixed candidate, Hoeffding's inequality gives
\begin{equation}
    \Pr(|\hat a(\alpha)-a(\alpha)|>\epsilon)\le2\exp(-2n_{\mathrm{cal}}\epsilon^2).
\end{equation}
A union bound over \(M\) candidates gives uniform deviation at level
\(\epsilon=\sqrt{\log(2M/\delta)/(2n_{\mathrm{cal}})}\). Let \(\alpha^\star=\arg\max_{\alpha}a(\alpha)\). Then
\begin{equation}
    a(\hat\alpha)\ge \hat a(\hat\alpha)-\epsilon
    \ge \hat a(\alpha^\star)-\epsilon
    \ge a(\alpha^\star)-2\epsilon.
\end{equation}
The margin rule compares the best CHOP mixture to the low-order candidate using the same uniformly controlled empirical accuracies, so the probability of enabling a worse high-order branch is bounded by the same calibration uncertainty unless the true gap lies within the margin band. This is why the method treats adaptive CHOP as a train-calibrated fallback rule, not as a universal high-order guarantee.
\end{proof}

\subsection{Combined Finite-Budget Error Decomposition}

The main theorems separate semantic error, sampling error, finite-shot error, sketch approximation, and high-order allocation. For a compact way to read them together, write \(\Pi\) for the schema used by the server and let \(\Pi^\star\) be the true semantic map. For a fixed class and query, the score error can be decomposed as
\begin{equation}
\begin{aligned}
|\hat s_c(z)-s_c^\star(z)|
\le&
\underbrace{E_{\mathrm{schema}}(\Pi,\Pi^\star)}_{\text{semantic bias}}
+\underbrace{E_{\mathrm{sample}}(n_c)}_{\text{finite examples}}
+\underbrace{E_{\mathrm{shot}}(S_{\min})}_{\text{finite readout}}\\
&+\underbrace{E_{\mathrm{rff}}(d)}_{\text{low-order sketch}}
+\underbrace{E_{\mathrm{hop}}(h)}_{\text{high-order sketch}}
+\underbrace{E_{\mathrm{cal}}(M,n_{\mathrm{cal}})}_{\text{adaptive gate}} .
\end{aligned}
\end{equation}
Under the assumptions in the main text, these terms may be chosen with high probability as
\begin{equation}
\begin{aligned}
    E_{\mathrm{sample}}&=O(n_c^{-1/2}),&
    E_{\mathrm{shot}}&=O(D^{1/2}S_{\min}^{-1/2}),\\
    E_{\mathrm{rff}}&=O(d^{-1/2}),&
    E_{\mathrm{hop}}&=O(h^{-1/2}),
\end{aligned}
\end{equation}
and
\begin{equation}
    E_{\mathrm{cal}}=
    O\!\left(\sqrt{\frac{\log M+\log(1/\delta)}{n_{\mathrm{cal}}}}\right).
\end{equation}
The important qualitative distinction is that \(E_{\mathrm{schema}}\) does not vanish with more samples, shots, or sketch dimensions unless the schema itself is corrected. This is the mathematical version of the paper's central information-fusion claim: communication and estimation can refine comparable evidence, but they cannot repair anonymous or wrongly named coordinates.
